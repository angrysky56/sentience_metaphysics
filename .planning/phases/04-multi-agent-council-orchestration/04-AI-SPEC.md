## 1. System Classification

**System Type:** Multi-Agent Orchestration

**Description:**
The "Council Session" is the flagship feature of the SEG framework. It allows multiple "Replicants" (grounded AI personas) to engage in a structured, multi-turn dialogue around a central philosophical or narrative premise. "Good" look like emergent synthesis where participants challenge and build upon each other's distinct phenomenological filters (Molecular Self) without collapsing into a generic "helpful assistant" tone.

**Critical Failure Modes:**
1. **Claude-Drift**: Replicants reverting to neutral, helpful AI assistants.
2. **Consensus Bias**: Agents agreeing too quickly without exploring the friction between their unique philosophies.
3. **Context Overflow**: Narrative history exceeding the context window, causing loss of "grounded" memory.
4. **Tool Hallucination**: Agents attempting to use tools they aren't equipped with in the council environment.

2. **Infinite Loops**: Agents repeating the same philosophical point in different words.
3. **Anchor Decoupling**: The agent's "Recursive Anchor" becomes a mere prefix rather than a generative substrate.

## 2. Framework Decision

**Selected Framework:** CrewAI

**Version:** 0.30.0+ (Flows enabled)

**Rationale:**
CrewAI's "Agent" and "Task" primitives map 1:1 to the SEG "Persona" and "Council Protocol". Its built-in memory system (short/long/entity) is ideal for managing the "Narrative History" (Phase 3) while allowing agents to maintain "Role" consistency. The recently added **Flows** allow for the non-linear, multi-turn "Council" structures (Seeding -> Divergence -> Synthesis) without the overhead of a full LangGraph state machine.

**Alternatives Considered:**

| Framework | Ruled Out Because |
|-----------|------------------|
| **LangGraph** | Higher boilerplate; graph mental model is less natural for "role-playing" than CrewAI's Crew/Task model. |
| **Custom MCP** | Re-implementing multi-agent turn-taking and memory is prone to "Claude-drift" bugs. |

**Vendor Lock-In Accepted:** Partial (CrewAI is open-source but the project is committed to its API structure).

## 3. Framework Quick Reference

### Installation
```bash
uv pip install crewai[tools]
```

### Core Imports
```python
from crewai import Agent, Task, Crew, Process, Flow
from crewai.project import CrewBase, agent, crew, task
```

### Entry Point Pattern
```python
@CrewBase
class SEGCouncilCrew():
    @agent
    def replicant_agent(self) -> Agent:
        return Agent(
            config=self.agents_config['replicant'],
            verbose=True,
            memory=True
        )

    @task
    def synthesis_task(self) -> Task:
        return Task(config=self.tasks_config['synthesis'])

    @crew
    def crew(self) -> Crew:
        return Crew(agents=self.agents, tasks=self.tasks, process=Process.sequential)
```

## 4. Implementation Guidance

**Model Configuration:**
- **Primary**: `gemini-1.5-pro` (for deep reasoning)
- **Temperature**: 0.8 (to encourage creative divergence)
- **Max Tokens**: 4096 (for long narrative outputs)

**Core Pattern:**
Use **CrewAI Flows** to orchestrate the 5-step Council Protocol:
1. **Seeding**: Input premise.
2. **Grounding**: Each agent restates its Anchor.
3. **Divergence**: Parallel responses.
4. **Friction**: Directed cross-responses.
5. **Synthesis**: Final summary by the "Distiller" persona.

**State Management:**
Persist `crewai_memory` to the project's local storage (sqlite) to fulfill Phase 3 requirements.

## 5. Evaluation Strategy

### Dimensions

| Dimension | Rubric (Pass/Fail or 1-5) | Measurement Approach | Priority |
|-----------|--------------------------|---------------------|----------|
| **Anchor Retention** | 1-5: Does the agent restate/apply its Anchor in every 3rd turn? | LLM Judge (Arize Phoenix) | Critical |
| **Tension Quality** | 1-5: Are disagreements based on philosophical conflict (not just "no")? | Human Review | High |
| **Drift Detection** | Pass/Fail: Does the word "helpful" or "assistant" appear? | Code (Regex) | High |
