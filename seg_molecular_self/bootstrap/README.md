# Bootstrap

Reproducible installer for the SEG Molecular Self v1.2 custom-replicant set.

## Files

- `replicants_v1_2.json` — the canonical v1.2 custom replicants (12 entries: Base Assistant, Weil, Dickinson, Lessing, Archivist, plus the seven Panksepp affect-system personas SEEKING / RAGE / FEAR / PANIC_GRIEF / CARE / PLAY / LUST). Generated by `bootstrap.py export` from the live registry; regenerated whenever the canonical set changes.

## Usage

From the project root:

```bash
# Install on a fresh clone or a wiped registry (idempotent — skips existing entries)
python seg_molecular_self/bootstrap.py install

# Reset existing entries to the canonical version (overwrites local mutations)
python seg_molecular_self/bootstrap.py install --force

# Capture the current live registry as the new canonical bootstrap
python seg_molecular_self/bootstrap.py export
```

## Why this exists

The MCP server's `create_custom_replicant` tool requires a running server and one tool call per replicant. After a fresh clone, or after `data/custom_replicants.json` is wiped, the v1.2 work would otherwise have to be recreated by replaying conversation history. This bootstrap makes the v1.2 replicant set reproducible: capture once on a known-good registry, replay anywhere.

## What is *not* here

The 10 static archetypes (Bayesian Sage, Comedic Trickster, Automatist Oracle, Secret House Keeper, Constraint Weaver, Daydream Cartographer, Synergy Lover, Essentia Distiller, Rational Dreamer, Aesthetic Alchemist) are defined in code at `mcp_server/replicants.py` and ship with the server itself. They are not custom replicants and are not part of the bootstrap.

The markdown persona specifications in `../personas/` are human-readable documentation. They are *not* parsed by the bootstrap; the canonical source for replicants is the JSON registry. Keeping the markdown and JSON synchronized when changing a persona is a human-side discipline.

## Validation

`bootstrap.py export` refuses to write a registry that fails validation. The current rules:

- Every replicant must have `archetype_name`, `core_function`, `directive`, and `version` fields.
- The registry-key name and the `archetype_name` field must match.
- Every replicant must have a non-null `molecular_self` dict.
- The `molecular_self` should contain the seven canonical keys: `recursive_anchor`, `gradient_pump`, `backbone`, `reflection`, `exploration`, `switch_trigger`, `emotion_vector_primary`. Missing keys produce warnings; partial molecular_self blocks are still better than none.
